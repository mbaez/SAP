Presentación

El Sistema de Gestión de Items de Proyecto pretende ser un sistema apto para el control eficiente de los componentes que componen cada fase de un proyecto de software. Previendo los posibles e inevitables cambios que surgen a través del tiempo de vida de un proyecto, el sistema estará preparado para proveer información suficiente al usuario para la correcta gestión de los componentes del proyecto. Además será capaz de determinar el impacto que tendrán estos cambios en las distintas fases.

  1. Modulo de Administración

    Este módulo es el encargado de la definición y preconfiguracion de estruturas dentro del sistema. Se podrán definir usuarios, roles, prpyectos, tipos de items y fases de acuerdo a los permisos correspondientes de los usuarios.

  2. Modulo de configuración

    En este módulo el sistema permitirá la gestión de las Lineas Base de un proyecto (Generación de Linea Base, Abrir Linea Base). Esta gestión se realizará por proyecto y estará a cargo de un usuario con los permisos correspondientes que generalmente será el Líder del mismo.

  3. Modulo de desarrollo

    Finalmente este módulo es el encargado de la administación de los items de un proyeto. El sistema permitirá la creación, modificaión y eliminación de los items en una fase especifica asi como la gestión de sus diferentes versiones a través su desarrollo.
